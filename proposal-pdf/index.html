AIエージェントとは ／ デモの見方

How it works

AIエージェントとは、指示を一つ一つ待つのではなく、状況（トリガー）を受けて自分で考え、段取りまで実行するAIです。 本デモは、その働きを目で見える形にしたもの。業種を切り替えて、自分ごととして体験できます。

従来のAI（チャット型）との違い
観点従来のチャットAIAIエージェント（本デモ）
きっかけ人が毎回 質問・指示する状況（トリガー）を受けて自分から動く
アウトプット単発の回答・文章メール・見積・請求・発注など"実行できる成果物"
作業範囲考えるのを手伝う（実行は人）分析→判断→作成まで一気通貫で段取り
人の役割指示し、自分で実行する最終承認するだけ（Human-in-the-Loop）
体験の流れ（3ステップ）
1
業種を選ぶ
医療／中小企業／個人事業主
2
トリガーを選ぶ
現場の出来事を選択 → AIが自律思考
3
人間が最終承認
「承認 / 修正」を押す。送信は承認後
思考プロセス（ReAct フレームワーク）
ステップ内容
① 状況分析（Thought）起きた問題と、予測されるリスク・影響を分析する
② 自律アクション（Action）指示されなくても「今やるべきこと」を自分で決定する
③ 実行内容（Execution）メール文面・見積書・請求書・発注書など具体物を作成する
④ 完了報告（Result）人へ事後報告し、最終確認・承認を求める
安心設計 ─ Human-in-the-Loop（人間が最終承認）
AIが担うのは「すぐ承認できる完成度の"案"を用意するところまで」。送信・発注・支払いなどの最終決定は必ず人間が行います。 だから安心して、面倒な準備だけを任せられます。次ページから、業種ごとの「できること」をご紹介します（各業種3シーン・計9シーン収録）。

エージェントの"中身" ─ ツールを使って自律処理

Tool Use

AIエージェントは、頭で考えるだけでなく、実際に社内の道具（ツール）を呼び出して作業します。デモでは、その一つ一つを「ツール実行ログ」としてライブ表示。"何を見て・何をしたか"がすべて見えるので、ブラックボックスになりません。

考えるだけでなく、道具を使って実行する
判断
どの道具を使うか決める
「カレンダーで空きを確認し、メールを作ろう」と自分で計画
実行
ツールを順に呼び出す
カレンダー / 台帳 / 会計 / メール…を1つずつ実行しログに残す
起案
結果を使って成果物に
取得した情報をもとに、メール・見積・発注書を生成

権限委譲 ─ どこまで"無人"にできるか

Autonomy Levels

Human-in-the-Loop は「固定」ではありません。業務のリスクと運用実績に応じて、AIに渡す権限を段階的に引き上げられます。低リスクは無人化、重要判断は人が承認と切り分けるのが現実的です。

自律度の4段階
L1
提案（現デモ）
AIが案を作り、毎回 人が承認してから実行
▲ 現在地
L2
例外承認
定型・低リスクは自動実行。例外だけ人へ
L3
ルール内 自動
決めた範囲（金額/相手/時間帯）内は完全自動
L4
完全自律
AIが実行、人は監督のみ（事後ログ確認）
引き上げの判断基準：① 同じ対応で十分な実績がある　② 失敗しても取り返せる（可逆）　③ 影響額・影響範囲が小さい　── 3つを満たすほど上の段階へ。
✅ 人を介さず任せられる領域（L3〜）
  • 消耗品・備品の定期補充発注（上限金額・指定業者・しきい値内）
  • 予約・日程の調整
  • リマインド送信（社内・既存顧客）
  • 書類・郵便・メールの自動仕分け
  • 在庫・期限・未入金の監視とアラート
  • FAQの自動一次回答
  • 定型レポートの自動生成・配信
共通点＝金額が小さい・可逆・定型。
⚠️ 必ず人が承認する領域（L1固定）
  • 人命・医療・安全に関わる判断（自動化しない）
  • 高額の発注・契約・支払いの確定
  • 対外的な謝罪・クレームの最終回答
  • 採用・人事・評価に関わる決定
  • 個人情報・機微情報の外部提供
共通点＝不可逆・対外・高リスク。
🔒 無人化を支える「ガードレール」
金額上限：1件・1日あたりの自動実行に上限 相手のホワイトリスト：登録済みの相手のみ自動送信 時間帯・件数制限：営業時間内・1日◯件まで 全操作の監査ログ：誰が・いつ・何をしたかを完全記録

設定した範囲を一歩でも超えた依頼は、自動で実行を停止し、人へエスカレーション。「任せきり」ではなく「柵の中で任せる」設計です。

"使う"から"自分で作る"へ ─ AIブートキャンプ

Bootcamp ①

ここまでご覧いただいたAIエージェントを、外注に頼らず自社で作れるようになるのが「関西ぱど AIブートキャンプ」です。 プログラミング未経験の非エンジニアでも、わずか2日間で"使える実務アプリ"をその場で構築。学ぶのはアプリの作り方ではなく、AIで業務そのものを変える具体的な施策です。

なぜ「経営者・幹部」が参加すべきか
業務のボトルネックを最も理解し、意思決定できる方（経営層・管理職）にこそご参加いただきます。トップ自らが「AIで何ができるか」を体感し、自社に必要な仕組みを形にすることで、 社内展開の判断が即決され、会社全体の事業スピードと生産性が根本から変わります。
カリキュラム ── 関西ぱどの"実践値"から生まれた、そのまま使える施策
01
Excel・手入力業務の自動化
日報・帳票・集計といった「名もなき手作業」をAIアプリ化。入力と転記の往復をなくす。
→ 入力工数を大幅圧縮
02
属人化業務の"仕組み化"
「あの人しかできない」業務を、誰でも回せる仕組みへ。引き継ぎリスクを解消。
→ 業務の標準化・脱属人化
03
高額SaaS・外注からの脱却
現場に合わない高額サブスクや外注を、自社内製アプリに置き換える。
→ ランニングコスト削減
04
現場に定着する内製アプリ
現場の声を即反映できるから、使われずに終わらない。改善が"回り続ける"。
→ 定着率・改善スピード向上
こんな企業様におすすめです
  • Excelや手入力業務が多く、属人化している
  • SaaSを導入したが自社に合わず、外注コストも高い
  • AI活用を次のステージへ進め、業務効率化や経営課題の解決に直結させたい
「アプリを作って終わり」ではありません。属人化していた手作業を仕組みに置き換え、ランニングコストを削減しながら、現場への定着率と改善スピードを引き上げる ── その全プロセスを、貴社の業務に当てはめて研修します。
